Frameworks

  • Frameworks provide boiler plate and components
  • The framework code is a single app where pages are different components
  • Page navigation is managed with routes by a router, e.g., http://localhost:8080/htmlplanets
  • A store is used for data that needs to be accessed by different pages
As an example the following figures are using a responsive Bootstrap gride layout.
Library vs. framework: you call library code in your code, framework code calls your code.
Components: components are reusable content, e.g., a page or part of a page.
Frameworks have a Model, a View, and a Controller. The Model contains the data, the view defines what is displayed, and the Controller implements the logic that controls how the data is used in the View. This is referred to as a Model, View, ViewModel (MVVM), or MV* or MVWhatever model.

3. Working vue-app tour

The starter app in vue-app was created using the Vue CLI and includes:
  • BootstrapVue. Bootstrap version for Vue. An alternative is Vue Material.
  • Vue Router used to map URLs to pages
  • Vuex to store data that is used throughout the app
  • axios promise based HTTP client
You can debug from within VS Code (see Debugging in VS Code) but usually using DevTools is sufficient.
The app features the following charts:
  1. HTML chart page showing COVID-19 data
  2. HTML chart page showing planets data
  3. D3 chart page showing COVID-19 data
  4. D3 chart page showing planets data
  5. Small multiples page uses a reusable D3 chart component showing COVID-19 data
We leverage the framework in various ways:
  • HTMLChart.vue and PlanetsChart.vue use the v-for directive to render an HTML bar chart.
  • BarChart.vue implements a reusable D3 bar chart. Arguments such as the title and the data are passed using props, e.g., <BarChart title="My title" :data="dataset"/>
