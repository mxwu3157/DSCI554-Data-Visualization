💡 Things to know about git repositories
  • Git snapshots are tracked using a SHA-1 hash, e.g., ca412e3a7968b81b247fa7ae40c2fd5bf7fb3308
  • The cloned git repository is found in the .git/ directory. All the files you see are are checked-out files.
  • When working in a team it's best to work with user branches and Pull Requests (PR)
  • Git commands you should know:
    • clone: fetch copy of remote
    • checkout: create working copy
    • add/rm: mark files to add
    • commit: save changes
    • pull: fetch changes from remote
    • push: upload changes to remote

Web basics

Node

Node is a JavaScript runtime environment. You can run node in the terminal: 
$ node              
Welcome to Node.js v16.15.0.
Type ".help" for more information.
> console.log('hello JavaScript')
hello JavaScript
undefined
> a = [1, 'b', {label: 'c', value: 3}, 2]
[ 1, 'b', { label: 'c', value: 3 }, 2 ]
> JSON.stringify(a)
'[1,"b",{"label":"c","value":3},2]'
>
When using node for data wrangling it is best to look for node packages (libraries). You can install node packages using npm -i <package-name>. The following demonstrates how to initialize a node repository using npm: 
# 🚨 do not copy-paste the prompt ($) and the comments #

$ npm init  #create package.json
$ npm install commander csv-parse  #fetches libs in node_modules and adds references in package.json
$ cat > myscript  #create myscript
#!/usr/bin/env node
console.log('hello myscript');
$ chmod +x myscript  #make executable
$ ./myscript  #run, create a bin and add to the PATH not to have to type ./
$ code -n .  #open current folder in VS Code to debug run
💡 Debug node scripts with console.log() statements and/or VS Code debugger (see launch.json)

HTML

HTML is stored in .html files. The index.html is the default file Web servers serve when the html file is not specified in the URL: if index.html is present it is returned and displayed in the browser, otherwise depending on how the server is configured a listing of the files and directories, or a 404 error page is displayed in the browser.
The following is an example of a minimal html file:
<html>
  <body>
    <h1>Header</h1>
    <h2>Sub-header</h2>

    <p>paragraph with text</p> 

    some bare text

    <p>
      Some text with a <a href="some_url">hyperlink</a>
    </p>
  </body>
</html>
💡 Things to know about HTML
  • HTML is about the content, cascading style sheets (CSS) is used to control how things look.
  • Only the contents of the body tag are visible!
  • Spaces, tabs and newlines outside of html elements are not rendered!
  • Browsers will not render malformed HTML: for most users no information it is better to show nothing than bogus information!
To debug HTML and javascript in Web pages use Chrome DevTools:
  1. Open index.html in Chrome
  2. Open DevTools: View → Developer → Javascript Console
  3. It is convenient to use a shortcut to open DevTools:
    DevTools shortcuts Mac Windows / Linux
    Open whatever panel you used last Command+Option+I F12 or Control+Shift+I
    Open the Console panel Command+Option+J Control+Shift+J
    Open the Elements panel Command+Shift+C or Command+Option+C Control+Shift+C

CSS

CSS stands for cascading style sheet. CSS is used to style HTML, e.g., using the style attribute: 
<div style="background-color: red;"">Hello<div>
Alternatively, one can store CSS in .css files that are then loaded in HTML with the link tag: 
<link rel="stylesheet" href="d3_line_chart.css">

Javascript basics

You can execute Javascript in Web pages using the script tag: 
<script type="text/javascript">
  console.log('hello Javascript')
</script>
Visualization libraries such as d3 are loaded from content delivery network (CDN) or installed with npm and included in the page using the script tag: 
<script src="../node_modules/d3/dist/d3.min.js"></script>
To use Javascript visualization libraries proceed as follows:
  1. Read the docs to understand what and where to include in the page (e.g., .css, js libs, and js code)
  2. Find an example with the same type of chart and implement it in the page
  3. Understand how the data is formatted in the example you implemented
  4. Format your data accordingly and replace the data in the example. Best is to load/hard code the data already formatted, i.e., do all needed data wrangling outside of the page!
  5. Customize labels, legends...
💡 Debug with console.log() and Chrome Console or VS Code debugger (see launch.json)

Markdown basics

Markdown is a markup language that is used in git repositories, notebooks, etc. for documenting.

  • Markdown is rendered to HTML: README.md in GitHub, Python and R notebooks.

  • Visual Studio Code let's you preview the Markdown

🚪 How to open lab files in the Browser

Whenever you see a node project (i.e., there is a project.json file in the folder, start by running npm install. To open the exercises (e.g., ex#.html and index.html file for the assignment) either:
  • Open the html file in VS code, then start Live Server from the status bar clicking Go Live
  • If the package.json contains a scripts section that includes a serve key with a browser-sync start... command, open the terminal in VS Code with View > Terminal and start a browser-sync server by executing npm run serve
